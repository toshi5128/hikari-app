## 🆕 4状態ワークフロー（v8.95-v8.116 完成）

### コンセプト
現場で空き家を認定 → 地番を後で入力 → 謄本取得 → 「DM送付対象」と「対象外（本人居住等）」に振り分け → 訪問対象だけを地図に表示

### 4状態判定ロジック（v8.114 + v8.116）

```js
const hasOwner = (j) => !!((j.owner && String(j.owner).trim()) || (j.ownerName && String(j.ownerName).trim()));

// 優先順位（上から判定）:
// 1. 🟡 地番待ち   = coordsManual && !address
// 2. 🚪 訪問OK     = coordsManual && address && hasOwner       （hasOwner 最優先）
// 3. 🚫 対象外     = coordsManual && address && !hasOwner && excluded
// 4. ⏳ 謄本待ち   = coordsManual && address && !hasOwner && !excluded
```

**重要**: `hasOwner` が真なら `excluded` に関係なく **🚪 訪問OK**（後から所有者が入った場合の救済）

**v8.116 リネーム**: 旧「📨 DM準備済」を「🚪 訪問OK」に変更（DM管理はスプレッドシート側に分離されたため）

### v8.115 表示ルール（既定で対象外を隠す）

- 通常の地図/一覧表示: **🚫 対象外を非表示**（excluded:true && quickFilter !== "excluded"）
- 「🚫 対象外」チップ選択時のみ表示
- カウンタは全件数表示（隠すのは地図/一覧の通常表示だけ）

### v8.108 + v8.114 マージロジック（Step 5 の挙動）

#### マッチング優先順位
1. **appId 一致**（取込「ID」列 == 既存 `id`）→ 既存ピンとフルマージ
2. **address 正規化一致** かつ既存 `coordsManual:true` → 所有者不問でフルマージ
3. **address+owner 一致**（従来）→ DM・住確のみ更新
4. どれもマッチしない → 新規追加

#### フルマージ時の挙動分岐（v8.114）

**A. 判定=対象外 で 所有者空 の場合**:
```js
next.excluded = true;
next.excludedReason = "本人居住";
// owner / ownerName / ownerAddr は触らない
// lat/lng/landmark/createdAt/source/coordsManual 維持
```
→ ピンは 🚫 対象外 へ移動。所有者情報は空のまま。

**B. 所有者あり（判定=送付 または判定列なし）の場合**:
```js
next.owner = it.owner;
next.ownerName = it.owner;  // 統一
next.ownerAddr = it.ownerAddr;
// もし excluded:true が残ってたら自動で解除（hasOwner 最優先）
if (next.excluded) {
  delete next.excluded;
  delete next.excludedReason;
}
```
→ ピンは 🚪 訪問OK へ移動。

#### 共有（同一appIdの複数行）の事前集約（v8.108 維持）
- 同じ `appId` の行を 1件に集約
- 所有者を `・` 連結（例: 黒澤真澄・黒澤健）
- 所有者住所を ` / ` 連結
- メモは `\n---\n` 連結

#### 既存保護（v8.108 維持）
- `coordsManual:true` ピンの `lat/lng` は絶対に上書きしない
- `landmark` / `createdAt` / `source` / `coordsManual` 維持
- DM・住確: 空/未では上書きしない
- 空欄補完のみ: `area`, `assignee`, `ownerContact`, `nextVisit`
- メモは追記式
- **訪問状況・初訪・備考はインポートで触らない**

### 🔴 [v8.114 追加・必読] CHECK 5: 「判定」列の運用

Claude Code が謄本取得した結果を「送付」「対象外」に振り分けて CSV に書く:

| 判定 | 所有者列 | アプリの挙動 |
|---|---|---|
| `送付` | **入れる**（必須） | → 🚪 訪問OK へ（従来通り） |
| `対象外` | **空のまま** | → 🚫 対象外 へ（owner/ownerAddr 触らない） |
| `送付` or 空 | 入ってる | → 🚪 訪問OK へ（判定列より hasOwner 優先） |
| `対象外` | 入ってる | → 🚪 訪問OK へ（判定列より hasOwner 優先・救済） |

**Claude Code の処理ロジック想定:**
```
for 各物件:
  謄本取得
  if 物件住所 != 所有者住所:
    判定列 = "送付"
    所有者列 = "<所有者氏名>"
    所有者住所列 = "<所有者住所>"
  else:  # 本人居住
    判定列 = "対象外"
    所有者列 = ""        # ← 重要: 空のまま
    所有者住所列 = ""
```

### 🔴 [現役・必読] CHECK 2: CSV/スプレッドシート列マッピング（v8.114更新版）

Claude Code が出力する集約スプレッドシートの列名:

| アプリ列名 | アプリ側フィールド | 備考 |
|---|---|---|
| `ID` | appId（マージキー） | **マージしたいなら必須**（CSV出力されたIDをそのまま出力に残す）|
| `物件住所` | address | 地番（市区町村から） |
| `所有者` | owner / ownerName（v8.108 で両方同時更新） | 共有時は1行ずつ別行に。対象外なら空 |
| `所有者住所` | ownerAddr | 対象外なら空 |
| `担当者` | assignee | |
| `緯度` | lat | 既存 coordsManual ピンの座標は保護される |
| `経度` | lng | 同上 |
| `メモ` | memo | 共有時は持分などを明記、「【共有者あり】」記載で v8.113 バッジ発動 |
| **`判定`** | judge（保存しない、マージ時に excluded 判定に使う）| **v8.114 新規**: `送付` / `対象外` |

**注意点:**
- CSV保存時は **UTF-8 (BOM付き)** または **Excel形式 (.xlsx)** を使う（Shift-JIS は文字化け）
- 「ID」列を必ず残す（マージのキー）
- 共有物件は同一 ID で複数行（アプリが自動集約して `所有者A・所有者B` 形式に）
- 対象外行は「ID + 物件住所 + 判定=対象外」だけでOK、所有者系列は空

### 🟡 [v8.115 影響] 既定での🚫対象外非表示

- 地上げ地図/一覧の通常表示時、`excluded:true` のピンは表示されない
- ただし `quickFilter === "excluded"`（チップタップ時）は表示される
- カウンタには件数が出る（pendingPinSummary は data.jiage 直接参照）
- 取り込み済みの対象外がアプリ画面に「見えない」状態 = 業務上は意図通り（DM対象でない＆訪問対象でないため）

### 🔴 [運用方針・要徹底] CHECK 6: 訪問記録はアプリ側で管理

**変化点（前と今）:**

| 項目 | 前（スプレッドシート主導） | 今（アプリ主導） |
|---|---|---|
| データの主 | スプレッドシート | アプリ（Supabase）|
| 訪問状況の記録場所 | スプレッドシートに記入 → 取込でアプリへ | **アプリで直接記録**（訪問記録モーダル）|
| マッチング | 物件住所＋所有者 | **ID（appId）**が最優先（v8.108）|
| 取込で更新される項目 | DM・住確のみ（v8.91） | 所有者情報・excluded フラグ（v8.108+v8.114）|
| 取込で保護される項目 | 訪問状況・初訪・備考・メモ | 訪問状況・座標・landmark・createdAt・memo・全部 |
| DM 管理 | アプリの DM 欄 | **スプレッドシート側で別途**（v8.116 で「DM準備済」→「訪問OK」に改名済み） |
| データの流れ | 1方向（スプレッドシート→アプリ） | **双方向**（アプリ⇄Claude Code）|

→ Claude Code 側でスプレッドシートを作るときは「**訪問状況や初訪日は含めない**」が原則。これらはアプリで直接管理する。

### 🔴 [厳守] 安全規則

1. **Supabaseへの書き込み前に必ずバックアップ**（`backup.yml` 手動実行 or `backups/`）
2. **既存データの構造を破壊する移行スクリプトを書かない**
3. **APP_VERSION の bump は必須**（数値比較されることを念頭に: `vM.m[.p]` 形式）
4. **`runGeocoding` の対象から `coordsManual: true` を除外**（既に実装済み）
5. **インポート時のマージ仕様を維持**: v8.108 + v8.114 のロジック
6. **同期ロジックを触る時は要注意**: tombstones / `_fieldUpdatedAt` / `protectedKeys` の関係
7. **データを多端末で扱うので、操作前に他端末の状態を考慮**

   - **入力**: アプリの「📥 謄本待ちCSV出力」が出すCSV（`tohon_machi_YYYY-MM-DD.csv`）
