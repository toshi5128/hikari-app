## 🌐 アプリ概要

- **名前**: ひかり不動産 PWA（社内 CRM / 業務支援アプリ）
- **対象業務**: 不動産仕入営業（地上げ＝空き家・所有者へのアタック）、仲介、在庫管理、積算、営業日報、メンバー間カレンダー共有
- **公開先**: https://toshi5128.github.io/hikari-app/
- **GitHubリポジトリ**: toshi5128/hikari-app
- **データ保存先**: Supabase（プロジェクト `ckcykbvislsjwvkizkqc`、テーブル `app_data`、id=`hikari_main`、`data` 列に JSON 全データ）

## 🛠 技術スタック

- **HTML 1ファイル構成**（`index.html`、現在約 16,000 行・900KB）
- **React 18** + JSX、`<script type="text/plain" id="hikari-jsx-src">` 内に記述

- **同期**: 15秒ポーリング、tombstone、`_fieldUpdatedAt`、保存前 RMW

## 🆕 「現地ピン」3段階ワークフロー（v8.95-v8.113 完成）

### コンセプト
現場で空き家を認定 → 地番を後で入力 → 謄本取得 → DM 送付フェーズへバトン

### ワークフロー全体（v8.111 で3段階可視化）

| Step | 担当 | 操作 | データ状態 | カウンタ |
|---|---|---|---|---|
| 1 | 現場担当 | 「📍 現在地で立てる」or「🗺 地図タップで立てる」 | `coordsManual:true, address:"", landmark, lat/lng:実測, assignee` | **🟡 地番待ち** |
| 2 | 確認担当 | 詳細→「✏️ 地番を入力して確定する」 | `address` 入力 → 通常色（未訪問）に | **⏳ 謄本待ち** |
| 3 | オペレーター | 「📥 謄本待ちCSV出力」（v8.112: ＋メニュー） | **謄本待ち（lookup_pending）だけ**を CSV 出力 | （変化なし） |
| 4 | **Claude Code** | CSV から謄本一括取得 → 所有者氏名/住所を集約スプレッドシートに付加 | （アプリ外） | （変化なし） |
| 5 | オペレーター | 「📥 Excel/CSVをインポート」 | v8.108 マージロジックで `owner`/`ownerName`/`ownerAddr` 追加 | **📨 DM準備済** へ移動 |
| 6 | 営業担当 | DM送付・訪問アタック | 既存業務フロー | （地上げ通常運用へ） |

### 3段階の判定ロジック（v8.111）

```js
const hasOwner = (j) => !!((j.owner && String(j.owner).trim()) || (j.ownerName && String(j.ownerName).trim()));

// 1. 🟡 地番待ち
coordsManual === true && (!address || address.trim() === "")

// 2. ⏳ 謄本待ち
coordsManual === true && address.trim() !== "" && !hasOwner(j)

// 3. 📨 DM準備済
coordsManual === true && address.trim() !== "" && hasOwner(j)
```

### v8.108 マージロジック（Step 5 の挙動）

優先順位:
1. **appId 一致**（取込「ID」列 == 既存 `id`）→ 既存ピンとフルマージ
2. **address 正規化一致** かつ既存 `coordsManual:true` → 所有者不問でフルマージ
3. **address+owner 一致**（従来）→ DM・住確のみ更新
4. どれもマッチしない → 新規追加

フルマージ時の保護:
- `coordsManual:true` ピンの `lat/lng` は絶対に上書きしない
- `landmark` / `createdAt` / `source` / `coordsManual` 維持
- DM・住確: 空/未では上書きしない
- `owner` と `ownerName` 両方に同じ値を入れる（地図ポップアップ・詳細・重複判定の表示統一）
- 空欄補完: `area`, `assignee`, `ownerContact`, `nextVisit`
- メモは追記式

共有（同一appIdの複数行）の事前集約:
- 同じ `appId` の行を 1件に集約
- 所有者を `・` 連結（例: 黒澤真澄・黒澤健）
- 所有者住所を ` / ` 連結
- メモは `\n---\n` 連結

### v8.112 CSV出力（謄本待ちのみ）

ボタン: 「📥 謄本待ちCSV出力（謄本取得用）」（＋メニュー内）

出力対象: **`coordsManual && address入り && !hasOwner`**（= ⏳ 謄本待ち のみ）

除外:
- 🟡 地番待ち（地番未入力）
- 📨 DM準備済（既に所有者情報あり = 謄本取得済み）

ファイル名: `tohon_machi_YYYY-MM-DD.csv`

CSV列構成:
```
ID / 作成日 / 地番 / 目印住所 / 担当者 / 緯度 / 経度 / ステータス / メモ
```

業務的意味:
- 既に謄本取得した物件を Claude Code に再投入しないので効率的
- 日次の運用で「新しく取得が必要な分だけ」を渡せる
- アラートに除外件数も表示（謄本待ち/DM準備済/地番待ちの件数）

### v8.113 共有者あり判定

判定: `ownerName` に「・」含む or `memo` に「【共有者あり】」含む

表示: 「👥 共有者あり」赤バッジ（`C.danger` = #ef4444）

表示場所:
- 地図ポップアップ（ピンタップ）
- 一覧（地図ビュー・リストビューのカード）
- 詳細BottomSheet（タイプバッジ section）

業務的意味:
- 謄本マージで `owner: "黒澤真澄・黒澤健"` のように複数人が連結されたケースを自動検知
- DM送付時に「共有物件なので両方に送る」を意識化
- 訪問時にも「複数所有者がいる」と理解した上で動ける

### v8.109 ヒント追加

現地ピン登録モーダルの目印住所(landmark)欄の直下に「Googleマップで場所を確認し、Googleマップ上の住所に直すのが望ましい」ヒント。
逆ジオコの精度限界を補い、ブルーマップでの地番照合精度を上げるため。

### 🔴 [現役] CHECK 2: CSV/スプレッドシート列マッピング

Claude Code が謄本取得結果をスプレッドシートに集約する際の列名:

| アプリ列名 | アプリ側フィールド | 備考 |
|---|---|---|
| `ID` | appId（マージキー） | **マージしたいなら必須** |
| `物件住所` | address | 地番（市区町村から） |
| `所有者` | owner / ownerName（v8.108 で両方同時更新） | 共有時は1行ずつ別行に |
| `所有者住所` | ownerAddr | |
| `担当者` | assignee | |
| `緯度` | lat | 既存 coordsManual ピンの座標は保護される |
| `経度` | lng | 同上 |
| `メモ` | memo | 共有時は持分などを明記、または「【共有者あり】」記載で v8.113 バッジ発動 |

**注意点:**
- CSV保存時は **UTF-8 (BOM付き)** または **Excel形式 (.xlsx)** を使う（Shift-JIS は文字化け）
- 「ID」列を必ず残す（マージのキー）
- 共有物件は同一 ID で複数行（アプリが自動集約して `所有者A・所有者B` 形式に）

### 🟡 [推奨] 入力 CSV のフィルタ

- 「📥 謄本待ちCSV出力」は **謄本未取得分だけ**を出力する仕様（v8.112）
- このCSVをそのまま Claude Code に渡せばOK
- すでに謄本取得済み（DM準備済）の物件は再処理されない

### 🔴 [厳守] 安全規則

1. **Supabaseへの書き込み前に必ずバックアップ**（`backup.yml` 手動実行 or `backups/`）
2. **既存データの構造を破壊する移行スクリプトを書かない**
3. **APP_VERSION の bump は必須**（数値比較されることを念頭に: `vM.m[.p]` 形式）
4. **`runGeocoding` の対象から `coordsManual: true` を除外**（既に実装済み）
5. **インポート時のマージ仕様を維持**: v8.108 で実装した優先順位ロジック
6. **同期ロジックを触る時は要注意**: tombstones / `_fieldUpdatedAt` / `protectedKeys` の関係
7. **データを多端末で扱うので、操作前に他端末の状態を考慮**
