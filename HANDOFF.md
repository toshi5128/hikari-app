## 🆕 「現地ピン」フルワークフロー（v8.95-v8.108 で完成）

### コンセプト
現場で空き家を認定 → 地番を後で入力 → 謄本取得 → DM 送付フェーズへバトン

### ワークフロー全体

| Step | 担当 | 操作 | データ変化 |
|---|---|---|---|
| 1 | 現場担当 | 「📍 現在地で立てる」or「🗺 地図タップで立てる」 | jiage に `coordsManual:true, address:"", landmark:"目印", lat/lng:実測, assignee:あなた`。**🟡 地番待ち** |
| 2 | 確認担当 | 詳細→「✏️ 地番を入力して確定する」 | `address` 入力 → 通常色（未訪問）に。営業tab「新規地上」+1 自動 |
| 3 | オペレーター | 「＋」→「📥 現地ピンCSV出力」 | 確定済みのみが CSV に。列: ID/作成日/地番/目印住所/担当者/緯度/経度/ステータス/メモ |
| 4 | **Claude Code** | CSV → 謄本一括取得 → 集約スプレッドシート作成 | 別システム（アプリ外） |
| 5 | オペレーター | 「＋」→「📥 Excel/CSVをインポート」 | jiage マージ（**現地ピンに所有者情報追加**） |
| 6 | 営業担当 | DM送付・訪問アタック | 既存業務フロー |

### v8.108 マージロジック（Step 5 の挙動）

優先順位:
1. **appId 一致**（取込「ID」列 == 既存 `id`）→ 既存ピンとフルマージ
2. **address 正規化一致** かつ既存 `coordsManual:true` → 所有者不問でフルマージ
3. **address+owner 一致**（従来）→ DM・住確のみ更新
4. どれもマッチしない → 新規追加

フルマージ時の保護:
- `coordsManual:true` ピンの `lat/lng` は絶対に上書きしない
- `landmark` / `createdAt` / `source` / `coordsManual` 維持
- DM・住確: 空/未では上書きしない
- `owner` と `ownerName` 両方に同じ値を入れる（地図ポップアップ・詳細・重複判定の表示統一）
- 空欄補完: `area`, `assignee`, `ownerContact`, `nextVisit`
- メモは追記式

共有（同一appIdの複数行）の事前集約:
- 同じ `appId` の行を 1件に集約
- 所有者を `・` 連結（例: 黒澤真澄・黒澤健）
- 所有者住所を ` / ` 連結
- メモは `\n---\n` 連結

### 実機テスト結果（v8.108、本日確認済み）

テスト内容:
- 現地ピン3件作成（地番入力済み）
- 4行のCSV（うち1ピンは共有で2行）をインポート

結果:
- ✅ **マージ動作正常**: 2件の現地ピンに所有者情報が正しく追加された
- ✅ **共有の集約 OK**: 同一 appId の2行が1レコードに `テスト 三郎・テスト 花子` で連結された
- ✅ **owner/ownerName 統一 OK**: 地図ポップアップにも詳細にも所有者名が表示される
- ✅ **実測座標保護 OK**: 既存の lat/lng が CSV の値で上書きされなかった
- ⚠️ **1件は新規追加**: 該当する現地ピンが事前に削除されていたため、appId/address どちらでもマッチせず → 新規レコードとして追加された（仕様通り）

#### 補足: 「籠原南」エントリの状態
- 元の現地ピン（ID 1780492700245）はユーザーが他作業中に誤って削除
- インポート時に該当 ID/address が見つからず → 新規追加
- 新規追加されたレコードは `coordsManual:false`（通常のインポート物件扱い）
- 業務的には所有者情報など必要なデータは揃っているため、運用上は問題なし
- 「未設定」エリアになっているのは、`籠原南` が JIAGE_AREAS（"熊谷"等）と部分一致しないため
  - 必要なら area を「熊谷」に手動で設定（籠原南は熊谷市内）

### 🔴 [現役] CHECK 2: CSV/スプレッドシート列マッピング

Claude Code が謄本取得結果をスプレッドシートに集約する際の列名:

| アプリ列名 | アプリ側フィールド | 備考 |
|---|---|---|
| `ID` | appId（マージキー） | **マージしたいなら必須** |
| `物件住所` | address | 地番（市区町村から） |
| `所有者` | owner / ownerName（v8.108 で両方同時更新） | 共有時は1行ずつ別行に |
| `所有者住所` | ownerAddr | |
| `担当者` | assignee | |
| `緯度` | lat | 既存 coordsManual ピンの座標は保護される |
| `経度` | lng | 同上 |
| `メモ` | memo | 共有時は持分などを明記 |

**注意点:**
- CSV保存時は **UTF-8 (BOM付き)** または **Excel形式 (.xlsx)** を使う（Shift-JIS は文字化け）
- 「ID」列を必ず残す（マージのキー）
- 共有物件は同一 ID で複数行（アプリが自動集約）

### 🔴 [厳守] 安全規則

1. **Supabaseへの書き込み前に必ずバックアップ**（`backup.yml` 手動実行 or `backups/`）
2. **既存データの構造を破壊する移行スクリプトを書かない**
3. **APP_VERSION の bump は必須**（数値比較されることを念頭に: `vM.m[.p]` 形式）
4. **`runGeocoding` の対象から `coordsManual: true` を除外**（既に実装済み）
5. **インポート時のマージ仕様を維持**: v8.108 で実装した優先順位ロジック
6. **同期ロジックを触る時は要注意**: tombstones / `_fieldUpdatedAt` / `protectedKeys` の関係
7. **データを多端末で扱うので、操作前に他端末の状態を考慮**
